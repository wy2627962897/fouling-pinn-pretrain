# 🚀 项目：换热器结垢热阻 AI 预测器（AI 加速版 → 可投稿升级版）

> 换热器结垢是化工行业排名前列的能耗损失来源——全球工业因换热器结垢造成的额外能耗成本每年高达数十亿美元。结垢是**无法避免、持续发生的动态过程**，机制复杂，数据稀缺——这正是 PINN 真正能发挥优势的场景。
>
> **本计划的特殊之处**：你不是一个人在做研究。Claude Code（以下简称 CC）作为 AI 协作者全程参与——负责机械劳动（写代码、搜文献、调格式、画图）、提供多角度批判（code review、prose critique）、加速迭代（几分钟完成传统需要几天的工作）。**你的角色是决策者与验证者**：判断方向、设计实验、验证物理正确性、撰写核心叙事。

---

## 研究定位：从演示项目升级为可投稿研究
> **核心判断**：合成数据 + Kern-Seaton + PINN 可以作为技术演示和入组材料，但若要投稿，必须补上真实数据、多机制模型、强 baseline、不确定性与清洗决策闭环。

- [ ] **[当前版本定位]** 明确当前成果属于"方法验证 / 技术报告 / 入组展示"：用 Kern-Seaton 合成数据证明物理约束在稀疏数据下的价值。
- [ ] **[投稿版本定位]** 将题目从"基于 PINN 的结垢预测"升级为"面向稀疏工业数据的换热器结垢热阻演化可解释预测与清洗决策"。
- [ ] **[核心风险识别]** 在报告中主动说明：如果用同一个物理模型造数据、再用同一个物理模型约束 PINN，容易被审稿人认为是自证循环。
- [ ] **[核心升级路线]** 把后续工作聚焦到四个关键词：真实数据、多机制、不确定性、决策闭环。
- [ ] **[目标期刊分层]** 先以 Applied Thermal Engineering / International Journal of Thermal Sciences / Case Studies in Thermal Engineering 为第一篇现实目标；高版本再考虑 Computers & Chemical Engineering、Energy、Applied Energy、International Journal of Heat and Mass Transfer、AIChE Journal。
- **✅ 达成此项：** 你知道当前版本能证明什么、不能证明什么，也知道论文升级真正缺哪几块。

## 当前优先 Sprint：先做没有企业数据时能做到的最硬证据（7-14 天）
> **核心思路**：现在不要先扩散到联系企业、清洗优化或大论文。第一优先级是把"公开真实曲线 + 多机制物理约束"做扎实，让项目先脱离纯合成数据。

- [ ] **[第 1 步：建立数据源台账]** 新建 `data_sources.md`，收集 15-20 篇可能包含原始数据、表格数据或可数字化曲线的论文。每篇只记录六项：题目、DOI/链接、结垢类型、图表编号、可提取变量、优先级。
- [ ] **[第 2 步：选 3 篇种子论文]** 先选最容易落地的三类数据：磷酸/蒸汽换热器工业数据、CaSO4 或 CaCO3 结晶结垢曲线、原油/有机物结垢曲线。不要一开始追求面面俱到。
- [ ] **[第 3 步：数字化第一批曲线]** 用 WebPlotDigitizer 或 Python 提取至少 10 条 $R_f(t)$ 或 $U(t)$ 曲线，保存原图、坐标轴范围、提取后的 CSV 和元数据。
- [ ] **[第 4 步：统一真实数据格式]** 建立 `data/real/metadata.csv` 与 `data/real/curves/*.csv`，字段至少包含 `source_id, curve_id, fouling_type, time, Rf_or_U, unit, Tw, velocity, concentration, notes`。
- [ ] **[第 5 步：做真实曲线基线拟合]** 对每条曲线先不训练神经网络，只拟合 Kern-Seaton、诱导期模型、阈值模型、清洗残余模型，看哪类曲线被哪种 ODE 解释得最好。
- [ ] **[第 6 步：建立多机制 ODE 注册表]** 在代码中把物理模型抽象为统一接口：`model_name -> parameters -> Rf(t) -> residual`，为后续 PINN loss 切换做准备。
- [ ] **[第 7 步：再接 PINN]** 等真实曲线和 ODE 注册表跑通后，再训练多约束 PINN。先做"每条真实曲线少量采样点训练、整条曲线预测"这个最清晰的实验。
- **✅ 达成此项：** 你拥有一批可追溯真实曲线、一个多机制 ODE 基线、一套可以接入 PINN 的物理约束接口。这是没有企业数据时最有论文价值的基础。

## 第一阶段：AI 协作 MVP —— 物理仿真（1-2 天）
> **核心思路**：传统方式是自己写代码实现模型。AI 时代的方式是**你说出需求，AI 生成代码，你验证物理正确性**。

- [ ] **[代码生成]** 向 CC 描述需求，让它生成 Kern-Seaton 模型（$dR_f/dt = \phi_d - \phi_r$）和 Ebert-Panchal 模型的 Python 实现。你不写循环和调包——你审核心：沉积项和剥离项的表达式对吗？参数的物理意义对吗？
- [ ] **[代码审查]** 用 `/code-review` 检查生成的仿真代码——不是查语法，是查物理：边界条件合理吗？量纲一致吗？渐近行为对吗（长时间后 $R_f$ 应该趋于稳定还是线性增长？）
- [ ] **[数据搜集]** 让 CC 搜索公开实验数据集（CaCO₃ 结晶结垢、CaSO₄ 沉淀结垢、颗粒沉积结垢、磷酸浓缩、原油预热器结垢），提取数据并整理成统一格式。找不到直接数据点时，从论文图表中数字化曲线并记录来源、工况、单位和不确定性。
- [ ] **[多机制模型]** 在 Kern-Seaton 之外加入 Ebert-Panchal 阈值模型、诱导期模型、清洗后残余热阻模型、污垢老化 aging 模型。先不追求复杂，至少能生成"渐近型 / 诱导期后突增 / 阈值不结垢 / 清洗循环"四类曲线。
- [ ] **[可视化]** 让 CC 生成 $R_f(t)$ 演化曲线——不仅画一条线，而是画多工况对比（不同流速/温度/浓度下结垢曲线的差异），标注清洗阈值和经济损失拐点。
- [ ] **[仿真器边界说明]** 给每个模型写清楚适用对象：结晶结垢、颗粒沉积、化学反应结垢、原油结焦、清洗后再结垢；避免把一个 ODE 包装成万能结垢机理。
- **✅ 达成此项：** 你有一个经过验证的物理仿真器，可以生成多种机制下的结垢曲线。这是后续 ML/PINN 工作的"数据工厂"，但不会替代真实数据验证。

## 第二阶段：AI 加速纯数据驱动 Baseline（1-2 天）
> **核心思路**：纯 DNN/LSTM 是你要对比的 baseline。AI 处理所有工程细节，你负责实验设计。

- [ ] **[数据工厂]** 用第一阶段的仿真器，让 CC 写数据生成器：10,000 组随机工况 → 对应结垢曲线，加高斯噪声模拟测量误差。你决定：噪声水平多高才接近真实工业数据？
- [ ] **[模型搭建]** 让 CC 搭训练流水线（MLP + LSTM + XGBoost/Random Forest + Gaussian Process + Neural ODE 作为候选），包括数据加载、训练循环、早停、checkpoint。你不写 boilerplate——你审核心：输入特征选对了吗？时间窗口多长合理？
- [ ] **[实验设计]** 你设计对比实验，CC 跑实验、出图表：
  - 标准工况：充足数据下的预测精度
  - 稀疏工况：仅有 20 个数据点时的表现（为 PINN 对比做准备）
  - 外推工况：训练到第 3 年，预测第 5 年
- [ ] **[失败分析]** 让 CC 分析 DNN/LSTM 在稀疏数据和外推场景下失败的原因——这恰好是 PINN 要解决的问题，也是你报告中最有力的论证。
- [ ] **[强 baseline 防守]** 记录每个 baseline 的调参空间和最佳结果，避免审稿人质疑"MLP 太弱，所以 PINN 赢得不公平"。
- **✅ 达成此项：** 纯数据驱动 baseline 完成，而且 baseline 足够强，能支撑论文级对比。

## 第三阶段：AI 协作 PINN —— 核心突破（2-4 天）
> **核心思路**：这是杀手锏。结垢数据稀缺（工厂不愿公开）、需要外推预测未来——传统 ML 搞不定，PINN 天生适合。AI 帮你实现和调参，你负责物理约束的设计和实验逻辑。

- [ ] **[架构设计]** 和 CC 讨论 PINN 架构：物理约束如何嵌入？Kern-Seaton ODE 写进 loss function 的哪一项？边界条件/初始条件如何编码？让 CC 用 DeepXDE 或原生 PyTorch 实现，你验证 ODE 残差项的正确性。
- [ ] **[轨迹配点修正]** 修复当前 PINN 外推失败的关键问题：物理配点必须沿同一工况轨迹采样，而不是每个 collocation point 随机换一套参数。每条轨迹固定 $(T_w, v, A, B, E_a)$，只沿时间 $t$ 计算 ODE 残差。
- [ ] **[硬约束架构]** 尝试把 $R_f(t)=R_{f,\infty}(1-\exp(-t/\tau))$ 或更一般的单调参数化形式嵌入网络输出层，让模型天然满足非负、单调、渐近行为。
- [ ] **[概念图]** 用 `concept-map` skill 画 PINN 架构图：神经网络层 + 物理约束层 + 数据损失层的关系。这张图会放进你的报告。
- [ ] **[关键实验]** 你设计三组实验，CC 跑通并出图：
  1. **稀疏数据优势**：仅 10-20 个数据点，PINN vs DNN 预测精度对比
  2. **外推能力**：训练到第 3 年，预测第 5 年，PINN vs DNN
  3. **物理一致性**：PINN 预测的 $R_f(t)$ 是否满足单调性和渐近行为（即使没有数据约束）
  > 这三组实验直接回答了导师最关心的问题：**"为什么要用 PINN 而不是普通神经网络？"**
- [ ] **[鲁棒性]** 让 CC 加不同水平的噪声测试 PINN 的稳定性。物理约束是否起到了正则化作用？
- [ ] **[消融实验]** 至少做四组消融：无物理损失、只加单调约束、完整 ODE 约束、硬约束输出层。用消融回答"到底是哪一部分带来提升"。
- [ ] **[代码审查]** 用 `/code-review` 全面检查 PINN 实现：loss function 权重合理吗？优化器选择对吗？有没有 silent bug？
- **✅ 达成此项：** 你有三张漂亮的对比图 + 一个经过验证的 PINN 模型。核心学术成果成型。

## 第六阶段：不确定性与清洗决策闭环（2-4 周）
> **核心思路**：工业现场最关心的不是一条漂亮预测曲线，而是"什么时候清洗最划算、风险有多大、还能安全运行多久"。

- [ ] **[不确定性量化]** 实现 Ensemble PINN、MC Dropout、Bayesian PINN 或 Conformal Prediction，输出 $R_f(t)$ 的置信区间，而不是单条曲线。
- [ ] **[RUL 估计]** 给定清洗阈值 $R_{f,crit}$，预测剩余可运行时间：$t_{clean}=\min\{t:R_f(t)\ge R_{f,crit}\}$，同时输出置信区间。
- [ ] **[经济目标函数]** 建立清洗决策成本模型：能耗损失 + 压降泵功 + 清洗费用 + 停机损失 + 产能损失。参数不确定时做敏感性分析。
- [ ] **[清洗周期优化]** 对比固定周期清洗、基于阈值清洗、基于 PINN 预测清洗、基于不确定性风险约束清洗四种策略。
- [ ] **[在线更新]** 模拟工业在线场景：每获得一个新测点就更新模型/后验分布，观察清洗时间建议如何变化。
- [ ] **[工程指标]** 除了 MAE/R²，增加工程指标：提前/延迟清洗天数、成本节省比例、误报警率、漏报警率、风险约束违反率。
- **✅ 达成此项：** 研究从"预测问题"升级为"预测 + 决策支持问题"，更适合 Applied Thermal Engineering、Computers & Chemical Engineering、Energy 类期刊。

## 第七阶段：更高版本论文元素（冲 Q1 / 高水平期刊）
> **核心思路**：高水平论文不能只说"我把 PINN 用到结垢上了"。它需要数据、机制、泛化、决策、可复现中至少两三项同时强。

- [ ] **[跨机制泛化]** 验证模型能处理至少两类结垢：例如 CaSO₄/CaCO₃ 结晶结垢 + 原油/有机物结垢，或结晶结垢 + 颗粒沉积。
- [ ] **[机制识别]** 让模型不仅预测 $R_f(t)$，还识别当前处于诱导期、快速沉积期、渐近期、老化期、清洗后恢复期中的哪一阶段。
- [ ] **[可解释参数]** 输出有物理意义的参数，如 $R_{f,\infty}$、时间常数 $\tau$、沉积速率、剥离系数、阈值温度/流速，而不是只输出黑箱预测。
- [ ] **[设备迁移]** 做 transfer learning / domain adaptation：从一个换热器或文献数据迁移到另一个换热器，证明少量新数据即可校准。
- [ ] **[高保真数据融合]** 尝试融合 CFD/HTRI/机理仿真和真实监测数据，用多保真学习降低真实数据需求。
- [ ] **[公开数据集]** 如果数据版权允许，发布一个整理好的结垢热阻演化 benchmark 数据集，包括原始曲线、元数据、提取脚本、baseline 结果。
- [ ] **[现场或半现场验证]** 最理想版本：拿到实验室循环水换热平台或企业匿名运行数据，做一次真实在线/离线验证。

## ⚠️ AI 协作的三大纪律

1. **物理正确性你说了算。** AI 能写出看起来合理的方程，但可能量纲错了、符号反了、边界条件不对。每行公式你都要问自己：这符合物理直觉吗？
2. **实验设计你说了算。** AI 可以跑 100 组实验，但"哪组实验能回答核心问题"由你决定。没有好的实验设计，再多数据也是噪音。
3. **核心叙事你说了算。** AI 能写流畅的文字，但"为什么这个研究值得做""结果意味着什么"只有你能说清楚。报告的灵魂是你的，不是 AI 的。
4. **真实数据优先级最高。** 合成数据适合 debug 和证明概念，但投稿时必须面对真实曲线、噪声、变量缺失和机制不匹配。
5. **不要让 PINN 成为口号。** 每个物理约束都要能回答一个审稿问题：它改善了稀疏数据、外推、物理一致性、不确定性，还是清洗决策？
